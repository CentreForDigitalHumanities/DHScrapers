How to harvest epidat records

epidat-syntax

baseurl
http://steinheim-institut.de/cgi-bin/epidat?

Each epidat record ID consists of three elements

  1. a unique code for the location ─ mandatory (consisting of three characters)
    followed by separator: "-"
  2. an unique ID ─ mandatory (consisting of digits)
    followed by separator: "-"
  3. format ─ optional
    • t | teip5
    • txt | plaintext

example: TEI XML
http://steinheim-institut.de/cgi-bin/epidat?id=hha-2338-t

If the format is not explicitly specified, then the default output format is html

example: HTML
http://steinheim-institut.de/cgi-bin/epidat?id=hha-2338

harvesting methods

  1. Method #1 provides access to all collections and their resources of the epidat collection.
    Each <resource> is part of a <collection> which is part of all epidat-<collections>.
  2. Method #2 allows more specific harvesting.
    It is also possible to harvest epidat records from a certain modification date
      3. example: Modification Date
      http://steinheim-institut.de/cgi-bin/epidat?sel=hha&format=x&function=changelog&changesSince=20141115
